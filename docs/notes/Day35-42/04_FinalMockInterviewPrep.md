# Final Mock Interview Prep

DAY 35-42: Final Mock Interview Preparation

## 1. Technical Story Structure

For project discussions, use:

Context:
- What product/problem?

Challenge:
- What was technically hard?

Decision:
- What options did you consider?

Trade-off:
- Why choose one approach?

Result:
- What improved? How measured?

Learning:
- What would you do differently?

## 2. Android Deep-Dive Topics To Rehearse

Kotlin:
- data classes, sealed classes, extension functions, null safety, generics variance.

Lifecycle:
- Activity/Fragment lifecycle, ViewModel survival, process death.

Concurrency:
- coroutines, structured concurrency, Flow, cancellation, thread safety.

Architecture:
- MVVM vs MVI, Clean Architecture, repositories, DI.

Testing:
- ViewModel coroutine tests, fakes/mocks, test strategy.

Performance:
- RecyclerView, layout optimization, memory leaks, ANR/jank.

## 3. Questions To Ask Interviewers

Strong questions:
- How does the Android team decide between shipping speed and technical debt?
- What architecture patterns are common in the codebase today?
- What are the most important reliability/performance challenges on Android?
- How are code reviews handled?
- What does success look like for this role in the first six months?

## 4. Communication Checklist

During coding:
- Clarify requirements.
- Confirm edge cases.
- Explain approach before coding.
- Think aloud on trade-offs.
- Start correct, then optimize.
- Test with examples.

During system/design discussion:
- Define scope.
- Identify constraints.
- Explain boundaries and ownership.
- Mention failure modes.
- Discuss testing and observability.

## 5. Final Review Drill

Before interview:
1. Rehearse 2 project deep-dives.
2. Solve 2 Kotlin medium problems timed.
3. Explain Activity lifecycle aloud in 2 minutes.
4. Explain coroutines vs threads aloud in 2 minutes.
5. Explain MVVM/MVI trade-off with an example.
6. Prepare 4 questions for interviewers.
